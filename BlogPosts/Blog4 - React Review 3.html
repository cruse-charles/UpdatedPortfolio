Reviewing React #3 - Props

Taking a look back on some React foundamentals - Props

Oct 7, 2024

What are props?

Short for properties, props are used to pass around information. React components are just JavaScript functions, and like all functions, they can receive arguments. In React, those arguments come in the form of a single props object. To better understand, consider props as 'instructions' that a parent passes down to a child.

We can pass primitive data types as well as objects and functions. We can also set a default prop in case nothing is passed down.

Passing Functions as Props

Props aren't limited to static data, you can also pass functions. When we pass a function from parent to child, we can call it in the child component. This pattern is essential for child-to-parent communication in React:

In this example, the Thruster can now use the startThrusters method that's actually defined in the Spaceship component. When the button in the Thruster component is clicked, it will call startThrusters, and print "Thrusters are ON" to the console.

Destructuring Props

This lets us unpack properties from objects or arrays right in our function signature. Instead of always writing props.name, props.status, etc., you can destructure props directly in the function parameter:

props.children

React gives us a special prop called children. Props.children allows developers to pass components as properties. This feature is particularly beneficial for creating 'wrapper' components.

Here, the Box component acts as a container or wrapper. It doesn't know what will be inside—it just renders whatever is passed to it via props.children. This is a core pattern for creating reusable components.

A More Interactive Example: Two-Way Data Flow

Let's now explore a more advanced example that shows how data and functions can flow in both directions between components.

Parent Component: This App component uses state to track a word and passes down handlers for updating and resetting that state.

Child Component This Word component uses the props it receives to: display the current word, update the parent's state when input changes, and reset the word when clicked.

Data Flow

From Parent to Child:

  1. The App component passes the current word (submittedWord) and the reset and change handlers (handleWordReset and handleWordChange) as props to the Word component.

From Child to Parent:

  1. When the input in Word changes, handleInputChange calls onWordChange with the new word.
  2. onWordChange is the handleWordChange function defined in App, which updates the submittedWord state in App.

Key Takeaways

  • Props are essential to making React components dynamic and reusable.
  • You can pass any type of data—including functions—as props.
  • props.children enables powerful composition patterns.
  • Destructuring props makes your code cleaner and easier to read.
  • Managing data flow via props is foundational to state management in React apps.
