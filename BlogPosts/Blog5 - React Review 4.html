Taking a look back on some React foundamentals - Event Handling and Forms

December 17, 2024

What is Event Handling?

React allows us to respond to user interactions like clicks, input changes, mouse movements, etc. These are called events. React handles events in a way that's very similar to native DOM handling, but with a few important differences:

  • Event names are camelCase instead of lowercase (onClick vs onclick)
  • You pass a function as the event handler

Handling Forms in React

React requires us to manage input state ourselves through controlled components. That means React components are in charge of storing and updating the input's value. A controlled input is an input element whose value is tied to a piece of React state.

The value prop is what allows us to control this field. We are essentially saying: "This input box should always display whatever is stored in the astronaut variable."

onChange listens for changes and updates the setAstronaut state variable using handleChange:

  • e is short for event - it's the synthetic event React passes when something happens (in this case, the input value changes).
  • e.target refers to the actual DOM element that triggered the event - the input element in this case.
  • e.target.value is the current text the user has typed into that input field.

Our handleSubmit function connected to the onSubmit property that is called when the form is submitted prevents the default form behavior and shows an alert with the astronaut's name. This pattern allows React to fully control the input's value, keeping everything in sync with your app's state.

Key Takeaways

  • Props are essential to making React components dynamic and reusable.
  • You can pass any type of data—including functions—as props.
  • props.children enables powerful composition patterns.
  • Destructuring props makes your code cleaner and easier to read.
  • Managing data flow via props is foundational to state management in React apps.
