Build With Me #1 - NextJS Project

See how I build a web app as I learn a new technology

January 6, 2025

Why I'm Building This

As someone passionate about teaching, I wanted to create a platform that helps others practice coding in an interactive way, while also helping me learn new things and refresh my skills. So, I decided to make Learn to Code — a web app that offers learners how to practice, learn, and improve on their coding skills.

My goal is to create an open and engaging platform to learn and reinforce programming fundamentals. I'll be building this project using Next.js, and I'll document everything I learn and implement along the way in this blog series.

As I continue to build on this, I hope to go from something that is functional for people to practice skills to something that can also work as a roadmap for people to really learn and improve in a structured way.

Why Next.js?

Next.js is a React-based framework that makes building full-stack web apps easier and more powerful. I chose it because I want to take a step forward with my coding skills. A few things I'll be learning more about while doing this project:

  • Server-side rendering (SSR) & Static site generation (SSG) - Ideal for both dynamic content (like coding challenges) and fast-loading static pages (like flashcards)
  • API routes — Lets me build backend logic (e.g., saving progress or serving problems) without spinning up a separate server
  • File-based routing — Simple and effective page management
  • Image optimization — Automatically optimizes images for faster loading

My Goals

My first goals are to create a few pages that can be used for general knowledge practice, DSA coding problems, and OOP testing principles. I want these to be fully functioning aspects of the website, allowing for customization, testing, and creativity for users.

After that, I'd like to focus on helping build some sort of learning path to help users improve, maybe even in career aspects as well instead of just coding.

Getting Started

First, I'd like to challenge what I think would be an easier part of the app, in terms of functionality, the flash cards. Here is what I'd like to build out:

  • Build out an engaging flashcard component
  • Create some sample mongodb documents for the flashcard
  • Retrieve the flashcards and display them
  • Create a filtering option for the flash cards

Immediately, I wanted to make the flashcards a bit fancier, just so they're slightly interactive with users rather than just boring cards. It took some research into different CSS properties, but I was able to get some core functionality into it. After getting the flipping animation to work, I used Vercel AI to clean up the styling and decide on a color theme.

Here's a peek of the Flashcard component when I was just getting functionality:


						"use client"

						import { useState } from "react"
					  
						interface FlashcardProps {
						  question: string;
						  answer: string;
						  category: string;
						}
					  
						export default function Flashcard({ question, answer, category }: FlashcardProps) {
						  const [isFlipped, setIsFlipped] = useState(false)
					  
						  const handleFlip = () => {
							setIsFlipped(!isFlipped)
						  }
					  
						  return (
							<div
							  className="h-72 w-full perspective-1000 cursor-pointer"
							  onClick={handleFlip}
							>
							  <div
								className={`relative w-full h-full duration-700 transform-style-3d ${
								  isFlipped ? "rotate-y-180" : ""
								}`}
							  >
								<div className="absolute w-full h-full backface-hidden">
								  <div>
									<h2>{question}</h2>
								  </div>
								</div>
								<div>
								  <div>
									<p>{answer}</p>
								  </div>
								</div>
							  </div>
							</div>
						  )
						}

What does this code do?

  • The outer div adds a perspective to create a 3D card flip effect.
  • The inner card container rotates 180° on the Y-axis using Tailwind utility classes like rotate-y-180.
  • backface-hidden: Ensures that only one side is visible at a time.

What I Learned

How to use the App Router (app/ folder) instead of a manual routing system like in base React. It was interesting to mark files as specific names like layout, page, and routes in order to get the functionality that I usually code out.

Any component that needs React hooks or browser interactions (like a useState) must be declared as "use client". This line goes at the top of the file. It was also interesting to see the differing speeds between the Server and Client side render when I added data on both sides to test out loading speed.
