A Quick NextJS Overview

Next.js is a React framework designed for building web applications, providing both server-side rendering (SSR) and static site generation (SSG) out of the box. Let's look into a few of the major benefits of using NextJS when creating an app.

What are the advantages of NextJS?

Server Side Rendering (SSR) - Instead of the server sending over HTML/JS and then rendering on the client, this is all rendered on the server and sent to the client already rendered, which is faster. This also helps with Search Engine Optimization (SEO). With the content already coming onto the page on the initial render, search engines can detect the content easily.

Simplified Features - NextJS also simplifies React development by offering features like automatic routing through file/folder structure, API routes that enable backend logic and eliminate the need for a separate backend server, and optimized loading strategies that help render content efficiently.

We'll breakdown a few of these...

SEO Optimization

With server-side rendering, we are able to get the actual content rendered on our server and then sent to the client as-is. With the rendering already being done on our server, it loads faster for our clients when sent to them.

Looking at our project page when we inspect the source code, we can see the content of a newly created NextJS app already. However, this content wouldn't be visible in just a normal React app. This is how NextJS optimizes SEO.

Furthermore, because we are able to work on a backend server through NextJS, we are able to build full-stack applications, not just front end or backend.

Routing

In a normal react app, we have to import 'react-router-dom' and wrap our app in a BrowserRouter tag. Then we must manually create each route/path and attach an element to the route created. This will also require creating an external backend for API routes.

In NextJS, creating folders within our app directory will create a route group. Layout files give a shared layout to this route group, things like sidebars or headers/footers, and are applied to all pages within. Page files will give us the main content for that URL.
